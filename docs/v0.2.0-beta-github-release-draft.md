# sepigs v0.2.0-beta.0

This is a beta release for acceptance testing. It is not production-stable and must not be treated as a stable or v1.0 release.

## Supported In This Beta

- HTTP proxy inbound with optional Basic Auth.
- SOCKS5 inbound with optional username/password auth.
- TCP direct, block, TCP relay, Shadowsocks, Trojan, and capability-boundary WireGuard outbounds.
- SOCKS5 UDP relay through the bounded UDP session manager.
- JSON/YAML config loading, route rules, rule-set expansion, GeoIP/GeoSite data loading, fake-IP, DNS cache, UDP DNS, and DoH architecture.
- Connection manager, resource limiter, timeout manager, statistics, Prometheus metrics, alert examples, benchmark tools, resumable soak runner, and security/documentation checks.
- Experimental local Web Dashboard with auth, rate limit, redaction, and local-only defaults.

## Experimental

- Web Dashboard.
- Shadowsocks/Trojan inbounds.
- WireGuard adapter boundary.
- TUN mock and packet parser.
- QUIC adapter boundary.
- Remote plugin factory RPC.

## Unsupported

- Production-stable native TUN.
- Native QUIC/Hysteria2.
- Active WireGuard forwarding.
- Shadowsocks UDP inbound.
- Remote subscription fetching.
- Stable public dashboard exposure.

## Security Defaults

- Default examples bind to `127.0.0.1`.
- Public HTTP/SOCKS listeners require authentication.
- Metrics and Dashboard default to local-only exposure.
- Dashboard tokens and proxy credentials are redacted by dry-run and security checks.
- Open-proxy style public configs without auth are rejected or documented as unsafe.

## Validation Status

- Local Phase 10 validation passed before tagging: lint, typecheck, tests, build, docs check, Web build, subscription dry-run, benchmark gate, short soak, 30-minute soak gate, security check, verification pack, and release dry-run.
- Phase 11 acceptance validation passed for remote refs, GitHub CI, manually dispatched release workflow, docs check, security check, 105-test suite, benchmark gate, 30-minute soak gate, and beta release dry-run.
- Latest Phase 11 30-minute soak gate: 465,784 successes, 239 recoverable fake-IP CONNECT read timeouts, 0.05129% error rate, final sockets/timers/listeners 0/0/0.
- Remote `main` and tag verification is tracked in `docs/v0.2.0-beta-remote-verification.md`.
- Full 24-hour soak remains an acceptance gate. A previous two-hour checkpointed segment of the 24h profile passed, but that is not a full 24h pass.
- GUI/mobile client sign-off remains manual and must not be marked verified until a tester records device, version, logs, and result.
- External Shadowsocks/Trojan interoperability depends on reference binaries; skipped rows are not equivalent to verified rows.

## Known Issues

- The 30-minute mixed soak gate observed recoverable fake-IP CONNECT timeouts during reload pressure below the beta threshold.
- Cross-component reload is not fully transactional yet.
- The Web Dashboard is beta/experimental and should stay local or behind trusted access controls.
- Full 24-hour soak, human client sign-off, and external reference implementation testing are still required before a production-stable claim.

## Quick Start

```bash
npm install
npm run build
cp examples/sepigs.safe.json sepigs.json
npm start -- --config sepigs.json
```

For validation:

```bash
npm run lint
npm run typecheck
npm test
npm run benchmark:gate
npm run soak:gate
npm run security:check
```

## Upgrade Notes

- Config version remains `1`.
- Existing v0.1.x examples should be reviewed against `docs/config.md` before reuse.
- Experimental features remain opt-in and should not be enabled for unattended public exposure.
