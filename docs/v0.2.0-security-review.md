# v0.2.0 Security Review

## Controls

- Dashboard token auth, loopback bind guard, rate limit, no-store responses, default-off CORS, and recursive config redaction.
- Subscription dry-run credential/key redaction and no remote code execution.
- Public unauthenticated HTTP/SOCKS listeners and public metrics binds are rejected by schema.
- UDP sessions have count/idle limits and shutdown cleanup.
- Plugin permissions/API version are checked; isolated runner crashes and timeouts are contained.
- `security:check` scans documentation/examples for private keys, local paths, non-placeholder credentials, public defaults, and redaction regressions.

## Residual Risks

- UDP can participate in amplification if operators expose an authenticated relay with weak credentials; keep listeners private and rate-limit upstream infrastructure.
- Fake-IP state reveals domain mappings and can cause DNS leakage if clients bypass the intended resolver path.
- Dashboard and proxy auth do not encrypt traffic; use a trusted host/network or an external secure tunnel.
- Metrics expose operational topology and must remain loopback or behind authentication.
- Worker/child plugin isolation is not an OS sandbox; in-process plugins are fully trusted.
- Shadowsocks/Trojan external implementation compatibility remains unsigned.

Decision: acceptable for a controlled beta with experimental features disabled by default. Not approved as a public open proxy or production-stable packet tunnel.
