# Protocol Compatibility

## Phase 8 Inbounds

Shadowsocks TCP inbound supports `aes-128-gcm`, `aes-256-gcm`, and `chacha20-ietf-poly1305`; each passes a real local encrypted stream through direct outbound. External reference-client verification remains pending, so status is experimental. Shadowsocks UDP inbound is unsupported.

Trojan TCP inbound validates password framing and can terminate TLS when certificate/key paths are configured. Plain mode exists only for local compatibility fixtures. External TLS/SNI client sign-off remains pending.

Phase 7 separates local protocol fixtures from external implementation interoperability. Reports are generated by:

```bash
npm run compat:matrix
```

Artifacts:

- [protocol-matrix.json](../reports/compat/protocol-matrix.json)
- [protocol-matrix.md](../reports/compat/protocol-matrix.md)

Current external summary for `v0.2.0-beta.0`: 0 verified, 11 blocked, and 0
failed. `sing-box` and `xray` are installed, but this release has no vetted
automatic launcher. No external interoperability result is inferred from local
fixtures.

## Shadowsocks

Verified by local fixture tests:

- sepigs outbound to local reference fixture.
- `aes-128-gcm`, `aes-256-gcm`, `chacha20-ietf-poly1305`.
- Evidence: `test/compat/shadowsocks-compat.test.ts`.

External reference implementation status:

- `sing-box` is installed at `/opt/homebrew/bin/sing-box`.
- `shadowsocks-rust` / `shadowsocks-libev` binaries are still missing.
- External cases are recorded as `blocked-with-reason` because this release has no vetted automatic launcher for the detected reference binary.
- No verified external Shadowsocks interoperability is claimed.

Unsupported:

- Reference client to sepigs Shadowsocks inbound. sepigs does not implement Shadowsocks inbound in Phase 7.

## Trojan

Verified by local fixture tests:

- sepigs outbound to local plain TCP reference fixture.
- Password hash line and TCP relay framing.
- Evidence: `test/compat/trojan-compat.test.ts`.

External reference implementation status:

- `sing-box` is installed at `/opt/homebrew/bin/sing-box`.
- `xray` is installed at `/opt/homebrew/bin/xray`.
- `trojan-go` is still missing.
- Wrong password, SNI/serverName, remote close, and large payload are recorded as `blocked-with-reason` until a vetted launcher can drive the detected reference binaries.
- No verified external Trojan interoperability is claimed.

Unsupported:

- Reference client to sepigs Trojan inbound. sepigs does not implement Trojan inbound in Phase 7.

## Reproduction Commands

```bash
npm run compat:shadowsocks
npm run compat:trojan
npm run compat:matrix
```

If a reference launcher is added later, rerun the matrix command and attach the refreshed `reports/compat/` artifacts. Do not mark external compatibility verified without the reference process logs and command output.
