# v0.2.0-beta.0 Security Summary

- Dashboard is disabled by default, loopback-only when enabled, token protected, rate limited, and recursively redacted.
- Metrics default to `127.0.0.1`; public metrics binds are rejected.
- Default HTTP and SOCKS listeners bind `127.0.0.1`; public HTTP/SOCKS requires authentication.
- Subscription dry-run redacts passwords, tokens, and key material.
- Release audit excludes `node_modules`, tests, profiles, reports, subscription fixtures, unrelated dist output, local paths, private keys, and files over 5 MiB.
- `.github/workflows` remains repository automation and is intentionally not part of the runtime archive. Its pending push is blocked by OAuth workflow permission, not removed from Git.

Residual risks remain: proxy authentication is not transport encryption, metrics have no built-in auth, fake-IP state reveals domain mappings, UDP exposure can amplify traffic, and JavaScript plugin isolation is not an OS sandbox. This release is suitable for controlled beta testing with experimental features disabled, not as an unaudited public proxy.
