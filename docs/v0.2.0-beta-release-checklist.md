- Required fix: use a PAT/OAuth credential with repository write plus workflow permission, or register an SSH key with write access and switch the remote to SSH.
- Tag: not created while `main` is absent from `origin`.

Credential repair commands for the user:

```bash
# HTTPS through GitHub CLI OAuth scopes
gh auth refresh -h github.com -s repo,workflow
git remote set-url origin https://github.com/Coilcap/sepigs.git
git push origin main

# Or SSH after adding a write-enabled key to the correct GitHub account/repository
git remote set-url origin git@github.com:Coilcap/sepigs.git
ssh -T git@github.com
git push origin main
```

After a successful push, restore HTTPS with `git remote set-url origin https://github.com/Coilcap/sepigs.git` if that is the preferred credential path. Do not remove workflow files to bypass the permission check.
