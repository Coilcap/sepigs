# sepigs v0.2.0 Client Verification Pack

Run sepigs with authenticated HTTP/SOCKS listeners. Desktop clients may use `127.0.0.1`; phones must replace `SEPIGS_HOST`, `SEPIGS_USERNAME`, and `SEPIGS_PASSWORD` with the trusted-LAN endpoint and inbound credentials. Keep firewall restrictions enabled and never expose an unauthenticated proxy publicly.

Test URL: `https://example.com/`. Success means the page loads through sepigs, one connection appears in Dashboard, and no credentials appear in logs. Revert by disabling the client proxy profile.
